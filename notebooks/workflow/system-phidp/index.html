System Differential Phase

System Differential Phase¶

Correct retrieval of System Differential Phase ΦDPsys\Phi_{DP}^{sys}ΦDPsys​ (aka system offset) is crucial for the accurate processing of all derived polarimetric products. In general, ΦDPsys\Phi_{DP}^{sys}ΦDPsys​ is assumed to be constant. However, it may exhibit azimuthal or elevational variations, for example due to antenna near-field effects.

Retrieval of ΦDPsys\Phi_{DP}^{sys}ΦDPsys​ is sometimes tedious, due to the contamination of the signal with clutter and other artifacts.

System Differential Phase ΦDPsys\Phi_{DP}^{sys}ΦDPsys​ via first precipitating bins¶

This is a most common algorithm. But there are several approaches. All use common ρHV\rho_{HV}ρHV​ filtering or other means of reducing unwanted artifacts in ΦDPsys\Phi_{DP}^{sys}ΦDPsys​.

  1. N consecutive radar bins with ρHV\rho_{HV}ρHV​ > threshold

  2. first N valid bins with ρHV\rho_{HV}ρHV​ > threshold (not necessarily consecutive)

Mask source data¶

Essential pre-step masking unwanted signal.

phimask = swp.uPhiDP.where(swp.RHOHV >= 0.9)

N consecutive valid radar bins¶

  1. It finds the first N consecutive precipitating bins per each ray

  2. and uses the median of these values to determine the offset per ray.

  3. If there are only a few of those radials (<30) per sweep we’ll use a default value from a previous sweep

  4. Sort the median values from 2. and calculate the median from the smallest 30 (default).

That value is considered ΦDPsys\Phi_{DP}^{sys}ΦDPsys​.

System Differential Phase ΦDPsys\Phi_{DP}^{sys}ΦDPsys​ via phase histogram¶

The idea behind is:

  • ΦDPsys\Phi_{DP}^{sys}ΦDPsys​ is constantly increasing

  • ΦDPsys\Phi_{DP}^{sys}ΦDPsys​ is inherently noisy

That means, the majority of phase measurements (precipitating bins) should lie around ΦDPsys\Phi_{DP}^{sys}ΦDPsys​. It’s relatively robust since it does not rely on finding precipitating bins in each ray or the like. Nevertheless, taking a pre-filtered phase as input should return similar results.
